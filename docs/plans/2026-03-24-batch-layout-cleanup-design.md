# 批处理上传布局精简设计

**日期：** 2026-03-24

**目标：** 优化批处理上传前端布局，将“推送文件名”从每个分组内部移到整体区域右侧，并删除对当前任务无帮助的辅助信息，突出“上传分组”和“命名”两个核心操作。

## 已确认需求

1. 采用推荐方案：左侧为分组文件上传区，右侧为统一的推送文件名侧栏。
2. 去除无关信息，重点保留必要操作、错误反馈和处理进度。
3. 在当前目录直接修改，不创建隔离 worktree。

## 当前问题

### 1. 信息层级混杂
当前每个分组卡片同时承载：
- 文件槽位上传
- 模板选择
- 自定义推送文件名
- 推荐文件名按钮
- 实际生效名称提示
- 分组状态
- 错误提示

这导致“上传操作”和“命名操作”混在同一块区域，纵向内容过长，浏览成本高。

### 2. 页面说明冗余
当前界面还有多层辅助说明：
- 分组区域顶部说明文案
- 每组推送文件名说明与生效说明
- 批处理面板中的任务摘要

这些信息在日常操作中不是核心决策信息，属于视觉噪音。

## 方案设计

### 整体布局
将 `CompositeUploadPanel` 主体改为两栏布局：
- **左侧主区域：** 分组上传卡片列表
- **右侧侧栏：** 统一展示每个分组的推送文件名输入框

在窄屏下两栏自动退化为上下布局，保证移动端可用性。

### 左侧保留内容
每个分组卡片保留：
- 分组标题
- 分组状态 Badge
- 删除按钮
- 文件槽位上传区
- 模板选择器
- 分组级错误提示
- 底部“新增一组”按钮

### 右侧保留内容
右侧侧栏按分组列出：
- 分组编号
- 推送文件名输入框
- 推荐名回填按钮（保留功能，但弱化说明文本）

右侧不再展示“实际生效名称”文案，输入框 placeholder 使用推荐名称承担提示作用。

## 删除项

### 从 `CompositeGroupEditor` 删除
- 顶部“组合分组”标题说明段
- 每组内部“当前组自定义文件名”模块
- “实际生效名称”说明文案

### 从 `CompositeUploadPanel` 删除
- `任务摘要` 模块
  - 完整组 / 部分组 / 空组统计
  - 当前可提交数量与上传文件数量说明

## 保留项
- 分组状态 Badge（帮助快速识别完成度）
- 全局错误提示
- 上传错误提示
- 提交按钮与处理中进度条
- 提交按钮中的有效任务数量

## 组件改造思路

### `CompositeGroupEditor`
职责收敛为“只负责分组上传编辑”：
- 文件槽位上传
- 模板选择
- 分组状态展示
- 分组错误
- 增删组

不再处理推送文件名 UI。

### `CompositeUploadPanel`
新增右侧命名侧栏：
- 统一接收 `groups`、`groupCustomPushNames`、`groupEffectivePushNames`
- 渲染每组命名输入
- 调用已有的 `onApplyGroupRecommendedName` 逻辑

## 响应式要求
- 桌面端：`lg` 及以上使用左右分栏
- 小屏：自动堆叠，命名区显示在分组区下方
- 输入区与文件卡片视觉风格保持现有卡片体系一致

## 测试策略
由于这是 UI 结构调整，优先补充 React 组件测试，验证：
1. 推送文件名输入不再出现在分组卡片内部。
2. 右侧侧栏会按分组渲染命名输入区域。
3. 任务摘要模块不再显示。
4. 既有分组错误提示和提交区仍可正常渲染。

## 风险与注意事项
- 右侧命名侧栏需要与分组数量动态同步，删除分组后输入项也要同步更新。
- 如果侧栏完全独立于分组卡片，需要确保用户仍能清晰知道“哪一个输入框对应哪一个分组”。
- 小屏布局退化时不能让页面变得更长更杂，需要保持标题和说明最小化。

## 结论
本次改造以“降低认知负担、提升批量操作效率”为目标，核心是把上传与命名拆开，并移除非关键辅助信息。这样能让用户在批处理场景下更聚焦于文件组织和结果命名两个主任务。
